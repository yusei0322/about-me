# 自己紹介ポートフォリオサイト｜課題要件 × 実サイト分析

> 京都デザイン＆テクノロジー専門学校「Web基礎」期末課題
> 対象：`https://<username>.github.io/about-me/` 系の公開ポートフォリオ群
> 分析対象：アクセス可能だった約90サイト（全114URL中、404・アクセス不可17件、SPA等で内容非表示のものを除く）

## 1. 課題として求められている条件

### 1-1. ページ要件
- TOPページ（`/`）
- 自己紹介ページ（`/about`）
- お問い合わせページ（`/contact`）
- その他コンテンツは自由（好きなものの紹介など）
- **合計5ページ以上**

### 1-2. 機能的要件
- ヘッダー・フッター・ナビゲーションが各ページにある
- レスポンシブ対応（メディアクエリの使用）
- トップページを含めてページ5枚以上
- 各ページにCSS・JavaScriptが適用され、テーマが統一されている
- お問い合わせフォームがある（送信機能は不要）
- 使用する画像・コンテンツはフリー素材か自身に著作権のあるもの
- 公序良俗から逸脱しない
- 独自性がある
- JavaScriptの実装が含まれている

### 1-3. 注意事項
- 顧客に納品するつもりで取り組む
- 他人の著作物を勝手に二次利用しない（オープンソース・パブリックドメインは可、署名要否に注意）
- ソースコードにも著作権がある
- 参考にするサイトは基本的に自由
- 生成AIの利用は歓迎（ただしレビューの有無が採点基準に含まれる）

### 1-4. 採点基準

#### 基本事項（すべて必須／未達・ふざけた内容は配点0）
- [ ] ヘッダー・フッター・ナビゲーションがある
- [ ] レスポンシブ対応している
- [ ] ページ枚数5枚以上（超過分は加点）
- [ ] テーマが統一されている
- [ ] 何らかのフォームがある
- [ ] 公序良俗から逸脱しない
- [ ] 独自性がある
- [ ] コンテンツが著作権上Webで利用可能
- [ ] 画像・動画がTOPに3つ以上（スライダー可）、各ページに最低1つ以上
- [ ] READMEが書かれている

#### 技術的な事項
- **HTML**：class命名規則／マークアップルール／エラーなし（W3C Validator）／AIレビュー
- **CSS**：リセットCSS使用／アンチパターン回避／エラーなし／AIレビュー
- **JavaScript**：各ページに1つ以上の実装／エラー放置なし／AIレビュー
- **その他**：ディレクトリ構造／画像形式（ラスター・ベクター）／ファイル名適切（日本語ファイル名NG）／AIレビュー

#### 加点要素
- ページ5枚を超える場合
- 未知の技術（実装の一部または全てに使用されていること。importのみは不可）
  - PHP／WordPress、Vue.js、React.js、Next.js、TypeScript、Alpine.js／Svelte／Solid.js、GSAP／Three.js

## 2. 90件の分析で見られた共通の「要素」

自己紹介コンテンツに実際に盛り込まれていた項目を、出現頻度の体感順に整理。

### 2-1. ほぼ全サイト共通
| 要素 | 補足 |
|---|---|
| 名前 | 本名またはもじったニックネーム |
| 趣味のジャンル別サブページ | ゲーム／アニメ・漫画／音楽／映画などを複数カテゴリに分割 |
| お問い合わせページ | メール・フォーム・SNSリンクのいずれか |

### 2-2. かなり多くのサイトで登場
| 要素 | 補足 |
|---|---|
| 出身地・現住所 | 都道府県レベルでの記載が中心 |
| 学校・専攻・学年 | ホワイトハッカー／スーパーITエンジニア／スーパーAIクリエイター／スーパーゲームクリエイター 各専攻 |
| 年齢・生年月日 | |
| ペット紹介 | 特に犬・猫。家族の一員として写真付きで丁寧に扱う傾向 |
| SNS導線 | Instagram／X／LINE等。QRコード掲載例も複数 |

### 2-3. 一定数見られた要素
- 血液型
- MBTI・16Personalitiesなどの性格診断結果
- 座右の銘・将来の夢／目標
- 「推し」ジャンルの深掘り（Vtuber、Hololive、K-POP、特定バンド等）
- 好きな食べ物・地元グルメ
- 旅行・海外研修の思い出（写真ログ形式）

### 2-4. 少数だが印象的だった要素
- 生い立ちの詳細タイムライン（出生地→幼稚園→小学校→中学校→高校→専門学校）
- 訪問国／地域数・滞在日数などの「自分データ」の数値化
- BGM・埋め込み動画・Spotify連携など音楽との紐付け
- 「話しかけてくれると嬉しいです」等、友達作りを意識した一言

## 3. 90件の分析で見られた「紹介のされ方」（表現スタイル）

| # | スタイル | 特徴 | 該当の傾向 |
|---|---|---|---|
| 1 | 基本データ型 | 「名前｜年齢｜出身地｜血液型」等をテーブル／箇条書きで淡々と提示 | 最多数派 |
| 2 | Q&A・ストーリーテリング型 | 自問自答形式や、生い立ちをエッセイ風に語る | 読み物として楽しませる意図 |
| 3 | ランキング・推し語り型 | 好きな作品を🥇🥈🥉で順位化、特定の対象を1ページ丸ごとファンサイト化 | 熱量高め |
| 4 | コンセプト演出型 | 架空の企業サイト（店・寺院）、SF端末UI、ゲームキャラ設定に見立てて自己紹介 | 課題の枠を超えた発想の転換 |
| 5 | ミニマル型 | TOPは名前とナビのみ、詳細は各サブページに委譲 | 構造のシンプルさ優先 |

> 全体傾向：単なる履歴書的自己紹介ではなく「好きなものを語ることで自分を表現する」設計が主流。趣味カテゴリ（ゲーム・アニメ・音楽・ペット）を独立ページとして厚めに作り込む点が共通していた。

## 4. 課題要件 × 実サイトの対応（要件がどう満たされていたか）

| 課題要件 | 実サイトでの現れ方 | 観察されたギャップ |
|---|---|---|
| 5ページ以上／ナビ完備 | 大半がTOP＋自己紹介＋趣味複数＋お問い合わせでページ数を確保。ナビゲーションはほぼ全サイトに存在 | 一部にページ未完成・リンク切れ |
| レスポンシブ対応 | `meta viewport` は大多数が設定済み。ハンバーガーメニュー実装例も複数 | コード上の確認に留まり、実表示は未検証 |
| テーマ統一 | 和風モノトーン、ネオン／SF、ゲーム世界観など統一感のある力作あり | テンプレ未編集で仮テキスト残存の例も |
| フォーム設置 | お問い合わせフォームは頻出。分野の複数選択・件名セレクト等の作り込み例も | 送信機能は要件通り任意 |
| TOPに画像3つ以上／各ページ1つ以上 | スライダー（Swiper等）で複数画像を satisfies する例が目立つ | 画像パスtypoで表示崩れの例あり |
| 独自性 | コンセプト演出型・API連携・自作SVG等で明確に差別化する上位層あり | 標準テンプレ構成の踏襲が多数派 |
| JavaScript実装 | スライダー、時刻表示、モーダル、言語切替、カウントアップ等 | 未編集のplaceholderが残るケースも |
| README | READMEを加点要素として明記し、制作意図・こだわりを記述した例あり | 記述の有無・充実度に差 |
| 加点技術 | Spotify API連携などの挑戦例。フレームワーク（Vue/React/Three.js等）の明確な使用は限定的 | 加点技術の活用余地は大きい |
